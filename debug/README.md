# Debug 脚本目录

本目录包含项目的调试和验证脚本，按功能模块组织。

## 目录结构

```
debug/
├── README.md                    # 本文件
├── video_info_collector/        # 视频信息收集器调试脚本
│   ├── debug_*.py              # 具体调试脚本
│   └── README.md               # 模块调试说明
└── [其他模块]/                  # 其他功能模块的调试脚本
```

## 使用原则

### 1. 调试脚本的作用
- **问题诊断**: 帮助定位和分析功能问题
- **流程验证**: 验证复杂业务逻辑的执行过程
- **数据检查**: 检查数据库状态和数据一致性
- **性能分析**: 分析性能瓶颈和优化点

### 2. 调试脚本规范
- **命名**: 使用 `debug_` 前缀，描述性命名
- **文档**: 每个脚本都应有清晰的注释说明用途
- **独立性**: 脚本应该能够独立运行，不依赖特定环境
- **安全性**: 不应修改生产数据，只进行读取和分析

### 3. 运行方式
```bash
# 从项目根目录运行
python debug/video_info_collector/debug_full_merge.py

# 或者作为模块运行
python -m debug.video_info_collector.debug_full_merge
```

## 注意事项

⚠️ **重要提醒**:
- Debug脚本仅用于开发和调试，不应在生产环境运行
- 运行前请确保了解脚本的作用和影响
- 如有疑问，请查看具体模块的README文档
